Anatomia do setup.py

Entenda como funciona o script de automação que criamos.

1 O Segredo da Venv

O principal desafio de scripts de automação em Python é rodar as dependências antes de tê-las instaladas. Usamos um truque inteligente: reiniciar o script.

Lógica do bootstrapper:

def main():
    # 1. Checa se já estamos na Venv
    if not is_venv():
        # Se não existir a pasta venv, cria
        if not VENV_DIR.exists():
            venv.create(VENV_DIR, with_pip=True)
        
        # O TRUQUE: Chama o script novamente, mas usando o Python da Venv
        venv_python = get_venv_python()
        subprocess.call([str(venv_python), __file__] + sys.argv[1:])
        return # Encerra o script "pai"

O script detecta que não está no ambiente virtual, cria ele se necessário, e então "chama a si mesmo" usando o executável do Python de dentro da Venv.

2 Instalação de Dependências

Agora rodando dentro da Venv, precisamos das bibliotecas. O script tenta importar; se falhar, instala. 

def install_dependencies():
    try:
        # Tenta usar as libs
        import flask
        import mysql.connector
    except ImportError:
        # Se falhar, usa o pip para instalar
        subprocess.check_call([sys.executable, "-m", "pip", "install"] + REQUIREMENTS)

3 Arquitetura: Setup vs Migration

Separamos as responsabilidades para manter o projeto organizado e seguro:

⚙️ setup.py

Cuida da infraestrutura.

  • Cria a venv
  • Instala dependências (`pip`)
  • Chama o `migration.py`
  • Inicia o `main.py`

🗄️ migration.py

Cuida dos dados.

  • Conecta no MySQL
  • Cria o Banco se não existir
  • Lê scripts da pasta `sql/`
  • Executa mudanças de schema
