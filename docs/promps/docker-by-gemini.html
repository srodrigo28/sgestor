Treinamento Docker Full-Stack

Dominando ambientes isolados para PHP, React e Python.

Por que o Docker é tão hypado?

O que ele resolve: A maior dor da programação é o conflito de ambientes. Você tem o PHP 8 instalado na sua máquina, mas pega um projeto antigo que exige o PHP 7.4. Sem o Docker, você teria que desinstalar ou usar gerenciadores complexos. O Docker cria "caixas isoladas" (containers) onde cada projeto tem seu próprio ecossistema, sem sujar o seu sistema operacional principal.

O Hype: O hype existe porque ele padroniza o ciclo de vida do software. O mesmo código, rodando na mesma versão da linguagem, vai se comportar de forma idêntica no notebook do desenvolvedor, no ambiente de testes e no servidor de produção (como uma VPS). Ele é a base da infraestrutura moderna e de arquiteturas de Micro-SaaS.

Entendendo Portas e Versões

Mapeamento de Portas (Host : Container)

Um container roda isolado. Para acessar um site dentro dele, precisamos fazer um "furo" nessa caixa. Usamos a sintaxe porta_do_seu_pc : porta_do_container.

ports:
  - "8080:80" # Acessa localhost:8080 para ver o Nginx (porta 80) do container
  - "3000:3000" # Típico para rodar o frontend React/Next.js

A importância das Versões (Tags)

Nunca use a tag latest (última versão) em produção. O seu projeto pode quebrar se a linguagem atualizar de surpresa. Sempre "trave" a versão exata que você está usando.

  • Ruim: image: python:latest
  • Bom: image: python:3.11.4

Os Tipos de Imagens no Docker Hub

Quando escrevemos um Dockerfile, sempre começamos com uma imagem base usando a instrução FROM. Existem diferentes categorias:

Imagens Oficiais

Mantidas pelos próprios criadores das linguagens (ex: php:8.2-fpm). São as mais seguras para se usar.

Alpine Linux

Versões minúsculas (ex: node:18-alpine). São ótimas para produção pois são muito leves (geralmente sob 50MB) e seguras, mas podem faltar algumas bibliotecas C padrão.

Ubuntu / Debian Based

Imagens mais parrudas. Pesam mais, mas são completas. Ideais quando seu projeto precisa compilar bibliotecas complexas (comum em Python).

Scratch

Uma imagem totalmente vazia. Usada geralmente para rodar binários compilados (como Go ou Rust) para obter o menor tamanho possível.

Como Rodar Multi-Stack

Veja como subiríamos um ecossistema com React, Python (API) e PHP (Admin) rodando juntos através do comando docker-compose up -d:

version: '3.8'

services:
  # 1. Frontend em React
  frontend:
    image: node:18-alpine
    working_dir: /app
    volumes:
      - ./react-app:/app
    command: npm start
    ports:
      - "3000:3000"

  # 2. Backend em Python (Flask/FastAPI)
  api_python:
    image: python:3.11
    working_dir: /app
    volumes:
      - ./python-api:/app
    command: python app.py
    ports:
      - "5000:5000"

  # 3. Painel Administrativo em PHP nativo
  admin_php:
    image: php:8.2-apache
    volumes:
      - ./php-admin:/var/www/html
    ports:
      - "8080:80"

Com este único arquivo, o Docker baixa o Node, o Python e o PHP, mapeia as pastas do seu computador para dentro dos containers e expõe as portas corretas. Tudo em segundos.
